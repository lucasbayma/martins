<objective>
Extract workspace and project lifecycle from `src/app.rs` into a new top-level module `src/workspace.rs`. This is the highest subprocess-integration surface in Phase 1 — create/archive/delete hit git CLI, tmux, filesystem, and state persistence — so correctness of call ordering matters more than any other plan.

Purpose: Pull ~400 lines of lifecycle code out of `src/app.rs`, leaving it with just the event-loop, construction, PTY-size sync, and helper surface. Phase 5 (Background Work Decoupling) will later make state-save async and this is the surface it will modify.
Output: New `src/workspace.rs` with every lifecycle mutation as a free async function. `App` methods become one-line delegators.
</objective>

## Trust Boundaries

Workspace lifecycle crosses three existing boundaries (unchanged by this refactor):
- App → git CLI (`git worktree add/remove`, via src/git/worktree.rs shelling out)
- App → tmux subprocess (new_session, kill_session, enforce_session_options, resize_session)
- App → filesystem (`std::fs::remove_dir_all` for worktree deletion)

This refactor does NOT change which process is invoked, the arguments, or the order of invocation — it only relocates the Rust code. No new trust boundary introduced.

| Boundary | Description |
|----------|-------------|
| (unchanged) | git CLI, tmux subprocess, filesystem — all pre-existing, call semantics preserved |
