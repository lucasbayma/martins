<objective>
Extract the central event-routing surface from `src/app.rs` into a new top-level module `src/events.rs`. This is the highest-risk plan in Phase 1 because event routing is where every user action enters the app; every path must behave identically.

Purpose: Pull ~700 lines out of `src/app.rs` — the largest single reduction in this phase. Gives Phase 2 (event loop rewire: dirty-flag + input-priority select) a single clean file to modify when wiring the new loop primitives.
Output: New `src/events.rs` with `handle_event`, `handle_key`, `handle_mouse`, `handle_scroll`, `handle_click`, `dispatch_action`, `handle_picker_click`, `apply_picker_outcome`, `activate_sidebar_item`, plus the geometry helpers used only by these paths. `App::handle_event` becomes a one-line delegator.
</objective>
