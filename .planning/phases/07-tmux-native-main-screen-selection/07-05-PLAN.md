<objective>
Land the keyboard precedence chains for Phase 7's tmux path: cmd+c (Tier 1 overlay → Tier 2 tmux buffer → Tier 3 SIGINT) and Esc (Tier 1 overlay clear → Tier 2 forward to delegating tmux session → Tier 3 fall through to PTY).

Purpose: Per 07-CONTEXT.md D-10/D-11/D-14 + 07-RESEARCH.md §Code Examples Example 2, the existing Phase 6 cmd+c branch (Tier 1 overlay copy + Tier 3 SIGINT) needs a NEW middle Tier 2: when overlay is empty AND we're delegating, spawn a blocking task that runs `tmux save-buffer - -t <session> | pbcopy`. The save-buffer subprocess takes ~5-15ms on M-series macOS — running it on the event thread is acceptable but moving it to `tokio::task::spawn_blocking` keeps cmd+c sub-50ms even on first invocation.

Esc gets a similar middle tier: when overlay is empty AND we're delegating AND `app.tmux_in_copy_mode()` is true (Plan 07-04 set the flag on Down forward), forward the `\x1b` byte to the tmux PTY. Tmux's vi-mode default would treat that as `clear-selection` — but the Plan 07-02 ensure_config override binds Escape to `cancel`, so a single Esc exits copy-mode entirely (D-14 single-press exit). After forwarding, we clear the local `tmux_in_copy_mode` flag.

Output: Modified handle_key cmd+c + Esc branches + 6 new tests (TM-CMDC-01..03, TM-ESC-01..03).
</objective>

//   Tier 1 — overlay selection → copy snapshot text via existing path.
//   Tier 2 — delegating session with non-empty tmux paste-buffer → spawn
//            `tmux save-buffer - -t <session> | pbcopy` off-thread.
//   Tier 3 — Terminal mode without either → SIGINT (0x03) to PTY.
if key.code == KeyCode::Char('c')
    && key.modifiers.contains(KeyModifiers::SUPER)
{
    // Tier 1 (Phase 6, unchanged): overlay selection.
    if let Some(sel) = &app.selection {
        if !sel.is_empty() {
            app.copy_selection_to_clipboard();
            // D-04: do NOT clear selection after copy.
            return;
        }
    }
    // Tier 2 (Phase 7 D-10): tmux paste-buffer when delegating.
    // Spawn off-thread to keep cmd+c sub-50ms even on cold tmux client.
    // save_buffer_to_pbcopy returns false silently when no buffer present
    // (RESEARCH §Subprocess Behavior); the user-visible effect of "no buffer"
    // is identical to "buffer was empty" — pbcopy receives nothing and the
    // existing clipboard contents are preserved.
    if app.active_session_delegates_to_tmux() {
        if let Some(session_name) = app.active_tmux_session_name() {
            tokio::task::spawn_blocking(move || {
                crate::tmux::save_buffer_to_pbcopy(&session_name);
            });
            return;
        }

    if app.mode == InputMode::Terminal {
        app.write_active_tab_input(&[0x03]);
        return;
