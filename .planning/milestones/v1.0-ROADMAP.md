# v1.0 — Fluidity Milestone (SHIPPED)

**Defined:** 2026-04-23
**Shipped:** 2026-04-27
**Duration:** 4 days
**Tag:** `v1.0`

## Milestone Goal

Make Martins feel native-terminal-fluid. Eliminate input lag, render-loop CPU burn, and background-work spikes; ship Ghostty-style text selection on the PTY main pane.

**Core Value:** Input-to-pixel responsiveness must feel like a native GPU-accelerated terminal (Ghostty/Alacritty baseline). If typing, clicking, switching workspaces, or selecting text in the PTY pane feels laggy, the app fails — users will reach for Ghostty/Alacritty + tmux directly.

## Headline Accomplishments

1. **Architectural split landed.** `src/app.rs` reduced from 2000+ LOC monolith to 436 LOC core. Event routing, modal dispatch, workspace lifecycle, and draw orchestration each live in single-responsibility modules. Every subsequent perf phase built on this surface without re-touching it.

2. **Dirty-flag rendering + input-priority select.** Idle CPU drops to near-zero (no more redraws when nothing changed). The `tokio::select!` loop now has a biased input-first branch so PTY output bursts and timers cannot starve keyboard/mouse events.

3. **Native-feeling PTY input + navigation.** Typing in the PTY pane renders each keystroke within one frame even under heavy agent log streaming. Sidebar/workspace/tab switching is instantaneous (NAV-01..04 operator-validated).

4. **Background work fully decoupled.** Git diff refresh is event-driven via debounced `notify` (~200ms) with a 30s safety-net fallback (no more 5s polling timer). State saves run async and never block input. File-watcher bursts coalesce.

5. **Dual-path PTY-pane text selection.** Phase 6 shipped Ghostty-style drag-select with REVERSED-XOR overlay + scroll_generation anchoring (highlight survives streaming PTY output). Phase 7 layered tmux's native copy-mode for non-mouse-app sessions (delegate path) — selection feels indistinguishable from running tmux directly in Ghostty. cmd+c follows 3-tier precedence (overlay → tmux paste-buffer → SIGINT). Esc follows 3-tier precedence. Tab-switch cancels outgoing tmux selection.

6. **145 tests passing, no regressions.** Baseline at milestone start was 0 (greenfield perf milestone, no existing test suite for fluidity surface); final 145 covering encoders, state machines, render passes, regression guards, and integration paths.

## Notable Architectural Patterns Established

- **`Arc<AtomicU64> scroll_generation` on PtySession** — anchored coordinate translation for selection stability across PTY scroll (Phase 6, extended in Phase 7)
- **`Arc<AtomicBool>` flags on PtySession** for cross-thread state (`tmux_in_copy_mode`, `tmux_drag_seen`, `tmux_gesture_delegating`)
- **3-tier keyboard precedence chains** (cmd+c, Esc) with explicit fall-through in `handle_key`
- **Conditional `handle_mouse` intercept gate** — overlay vs delegate decision made per-event, latched per-gesture
- **`save_state_spawn` primitive** — fire-and-forget async state persistence; 13 hot-path call-sites migrated
- **Override-only tmux.conf philosophy** — bind only the lines that diverge from tmux 3.6a defaults; never re-bind defaults (drift-prone)
- **Env-var-gated diagnostic instrumentation** (`MARTINS_MOUSE_DEBUG`) — zero-cost when off, available for future re-investigation
